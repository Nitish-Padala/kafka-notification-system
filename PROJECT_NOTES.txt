# KAFKA NOTIFICATION SYSTEM - CHEAT SHEET

## PHASE 1: SETUP
- Tech Stack: Java 17, Spring Boot 3, Apache Kafka (Docker).
- Docker: Use docker-compose for Zookeeper & Kafka (Local Broker).
- Scaffolding: Spring Initializr -> Web, Kafka, Lombok.

## PHASE 2: KAFKA PRODUCER
- Workflow: API Request -> Controller -> Producer -> Kafka Topic.
- DTO: NotificationRequest (recipient, message, channel).
- Serialization: JsonSerializer converts Java objects to JSON bytes for Kafka.
- Controller: @RestController + @PostMapping for receiving requests.

## PHASE 3: EMAIL DELIVERY
- Architecture: Service Pattern (Interface -> Implementation).
- Loose Coupling: Consumers depend on NotificationService (Interface).
- Email Engine: spring-boot-starter-mail uses Jakarta Mail (SMTP protocol).
- Auto-Config: spring.mail.* properties automatically configure JavaMailSender bean.

## PHASE 4: PUSH NOTIFICATIONS
- Engine: Firebase Cloud Messaging (FCM).
- Config: Service Account JSON key used for server-side authentication.
- Device Token: Push requires a unique Registration Token per device.
- Service Worker: `firebase-messaging-sw.js` is required for browser background delivery.

## PHASE 5: SMS DELIVERY
- Engine: Twilio SDK.
- Authentication: Account SID and Auth Token.
- Validation: Recipient must be in E.164 format (e.g., +91...).
- Trial Mode: Restricted to verified phone numbers.

## SECURITY & GIT
- Secrets: Never commit keys to Git. Use `.gitignore` and `application.properties.example`.
- Git Strategy: Push to feature branches, merge to main via Pull Requests.
